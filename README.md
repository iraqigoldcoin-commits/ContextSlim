# ContextSlim

**ContextSlim** is a zero-config CLI tool that scans your project, strips the noise, and produces a single, token-optimized `llm_context.md` file ‚Äî purpose-built for feeding clean context to LLMs like [Claude Code](https://claude.ai/code), ChatGPT, and Gemini.

Stop copy-pasting files. Stop burning context tokens on lock files, build artifacts, and blank lines. Run one command and hand an LLM exactly what it needs to understand your entire codebase.

---

## Why ContextSlim?

| Problem | Without ContextSlim | With ContextSlim |
|---|---|---|
| Context pollution | LLM sees `node_modules`, lock files, images | Only source files that matter |
| Token waste | Raw code with excessive blank lines | Whitespace-compressed, trim output |
| Manual work | Copy-pasting files one by one | One command generates everything |
| Disorientation | LLM lacks project structure | Tree + content in a single file |

---

## Features

- **Smart directory tree** ‚Äî ASCII project tree with configurable depth, sorted directories-first.
- **Intelligent file filtering** ‚Äî ignores `node_modules`, `dist`, `.git`, lock files, images, binaries, and more out of the box.
- **Whitespace compression** ‚Äî strips trailing spaces and collapses consecutive blank lines to save tokens.
- **Language-aware code blocks** ‚Äî every file is wrapped in a fenced code block with the correct language identifier.
- **Size guardrails** ‚Äî automatically skips files above a configurable KB threshold.
- **Per-project config** ‚Äî drop a `.contextslim.json` in your project root for persistent, team-shared settings.
- **Stats mode** ‚Äî see file count, skipped count, total characters, and estimated token count after generation.
- **Zero config required** ‚Äî works out of the box on any project.

## Usage

```bash
contextslim [directory] [options]
```

### Examples

```bash
# Scan current directory ‚Üí llm_context.md
contextslim .

# Scan a specific project
contextslim ~/projects/my-api

# Custom output file name
contextslim . --output context-for-claude.md

# Tree only (no file contents)
contextslim . --no-content

# Show stats after generation
contextslim . --stats

# Limit tree/file depth and file size
contextslim . --max-depth 5 --max-size 200

# Add custom ignore patterns
contextslim . --ignore "**/__tests__/**" "*.test.ts"

# Include extra patterns that would otherwise be ignored
contextslim . --include "*.env.example"
```

### All Options

| Flag | Default | Description |
|---|---|---|
| `[directory]` | `.` | Target project directory |
| `-o, --output <file>` | `llm_context.md` | Output file name |
| `-d, --max-depth <n>` | `10` | Max directory depth |
| `-s, --max-size <kb>` | `500` | Skip files larger than N KB |
| `--no-tree` | ‚Äî | Omit the directory tree |
| `--no-content` | ‚Äî | Omit file contents (tree only) |
| `--include <globs...>` | ‚Äî | Extra patterns to include |
| `--ignore <globs...>` | ‚Äî | Extra patterns to ignore |
| `--stats` | ‚Äî | Print stats after generation |

## Per-Project Config (`.contextslim.json`)

Place a `.contextslim.json` file in your project root to define defaults. CLI flags always take precedence.

```json
{
  "output": "context.md",
  "maxDepth": 6,
  "maxSizeKb": 250,
  "stats": true,
  "ignore": [
    "**/__tests__/**",
    "**/*.test.ts",
    "**/*.spec.ts",
    "docs/"
  ],
  "include": [
    ".env.example"
  ]
}
```

## What Gets Ignored by Default

ContextSlim ships with a comprehensive built-in ignore list:

- **VCS**: `.git`, `.svn`, `.hg`
- **Dependencies**: `node_modules`, `vendor`, `.venv`, `__pycache__`
- **Build outputs**: `dist`, `build`, `.next`, `.nuxt`, `coverage`
- **Lock files**: `package-lock.json`, `yarn.lock`, `pnpm-lock.yaml`, etc.
- **Binary & media**: images, fonts, audio, video, archives, executables
- **IDE & OS**: `.DS_Store`, `.idea`, `.vscode`, `Thumbs.db`
- **Secrets**: `.env`, `*.pem`, `*.key`
- **Generated assets**: `*.min.js`, `*.map`, `*.log`

## Workflow with Claude Code

1. Run `contextslim .` in your project root.
2. Open `llm_context.md`.
3. Paste the entire contents into your Claude Code conversation (or attach the file).
4. Claude now has a complete, clean picture of your project without wasting tokens.

For large codebases, combine with `--max-depth` and `--ignore` to further narrow scope.
